# 📘 การบ้าน 5: User Requirement Specification (URS)
## กรณีศึกษา: ระบบ POS ของ Golden Place

---------------------------------------------------------

## Section 1: System Overview

ระบบ POS (Point of Sale) ของ Golden Place เป็นระบบที่ใช้สนับสนุนการทำงานของพนักงานคิดเงินบริเวณจุดชำระเงิน โดยระบบทำหน้าที่หลักในการสแกนบาร์โค้ดสินค้า แสดงชื่อและราคาสินค้าแต่ละรายการบนหน้าจอ คำนวณยอดรวมทั้งหมดโดยอัตโนมัติ รับการชำระเงินจากลูกค้าทั้งแบบเงินสดและผ่านบัตร/QR Code และพิมพ์ใบเสร็จรับเงินให้ลูกค้าหลังทำรายการเสร็จสิ้น นอกจากนี้ Golden Place ยังมีระบบสมาชิกสะสมแต้มที่เชื่อมกับการซื้อสินค้าหน้าร้าน จึงคาดว่าระบบ POS ต้องรองรับการผูกบัญชีสมาชิกและคำนวณแต้มสะสมในขั้นตอนคิดเงินด้วย ระบบนี้ถูกออกแบบมาเพื่อช่วยลดความผิดพลาดในการคำนวณราคาด้วยมือ และช่วยให้กระบวนการชำระเงินหน้าร้านเป็นไปอย่างรวดเร็วและถูกต้อง

## Section 2: User Roles

1. **พนักงานคิดเงิน (Cashier)** — ผู้ใช้งานหลักที่สแกนสินค้า ดำเนินการคิดเงิน และรับชำระเงินจากลูกค้า
2. **ลูกค้า (Customer)** — ผู้รับบริการที่นำสินค้ามาชำระเงิน และเป็นผู้ตรวจสอบยอดรวมและรับใบเสร็จ

## Section 3: Functional Requirements

1. ระบบควรสามารถสแกนสินค้าด้วยเครื่องอ่านบาร์โค้ดและดึงข้อมูลราคาสินค้าได้โดยอัตโนมัติ
2. ระบบควรสามารถแสดงชื่อสินค้าและราคาต่อรายการบนหน้าจอให้ลูกค้าเห็นได้ทันทีหลังสแกน
3. ระบบควรสามารถคำนวณยอดรวมของสินค้าทั้งหมดในตะกร้าได้โดยอัตโนมัติ
4. ระบบควรสามารถรับการชำระเงินได้หลายช่องทาง เช่น เงินสด บัตรเครดิต/เดบิต และ QR Code
5. ระบบควรสามารถพิมพ์ใบเสร็จรับเงินที่แสดงรายการสินค้า ราคา และยอดรวมให้ลูกค้าได้
6. ระบบควรสามารถแก้ไขหรือยกเลิกรายการสินค้าที่สแกนผิดพลาดก่อนยืนยันการชำระเงินได้
7. ระบบควรสามารถคำนวณเงินทอนให้กับลูกค้าในกรณีชำระด้วยเงินสดได้อัตโนมัติ
8. ระบบควรสามารถบันทึกและคำนวณแต้มสะสมของสมาชิกจากยอดซื้อในแต่ละครั้งได้

## Section 4: Non-Functional Requirements

1. ระบบควรทำงานได้อย่างรวดเร็ว เพื่อไม่ให้ลูกค้าต้องรอคิวนานในช่วงเวลาที่มีลูกค้าหนาแน่น
2. ระบบควรมีหน้าจอที่ใช้งานง่ายและอ่านง่าย เพื่อให้พนักงานใหม่สามารถเรียนรู้การใช้งานได้อย่างรวดเร็ว
3. ระบบไม่ควรค้างหรือหยุดทำงาน (crash) ระหว่างการทำรายการ เพื่อป้องกันความล่าช้าและความไม่พอใจของลูกค้า
4. ระบบควรมีความแม่นยำในการคำนวณราคาและเงินทอนสูง เพื่อลดข้อผิดพลาดที่กระทบต่อความน่าเชื่อถือของร้าน

## Section 5: Observations & Issues

- **ร้านมีคนแน่นและคึกคักเป็นพิเศษช่วงวันหยุดสุดสัปดาห์**: จากรีวิวลูกค้าที่แชร์ประสบการณ์ไว้ทางออนไลน์ (เช่น สาขาสะพานสูง) ระบุว่าช่วงเสาร์-อาทิตย์มีลูกค้ามาเดินซื้อของกันหนาแน่นเป็นพิเศษ ซึ่งบ่งชี้ว่าจุดคิดเงินน่าจะต้องรองรับปริมาณธุรกรรมที่พุ่งสูงในช่วงเวลานี้ ระบบจึงควรมีความเร็วในการประมวลผลที่เพียงพอไม่ให้เกิดคอขวดที่แคชเชียร์
- **มีระบบสมาชิกสะสมแต้มที่ผูกกับยอดซื้อ**: ข้อมูลจากเว็บไซต์และบทความแนะนำร้านระบุว่า Golden Place มีระบบสมาชิกที่ให้แต้มสะสมตามยอดซื้อเพื่อแลกส่วนลดในบิลถัดไป ซึ่งหมายความว่าหน้าจอ POS ต้องมีขั้นตอนเพิ่มเติมสำหรับกรอก/สแกนบัตรสมาชิกและคำนวณแต้ม ซึ่งอาจเป็นอีกจุดที่ทำให้ขั้นตอนคิดเงินใช้เวลานานขึ้นถ้าออกแบบ flow ไม่ดี
- **ข้อจำกัดของข้อมูล**: ไม่พบรีวิวหรือโพสต์ออนไลน์ที่พูดถึงรายละเอียดการทำงานของ POS หรือปัญหาที่จุดคิดเงินโดยตรง (ส่วนใหญ่รีวิวเน้นพูดถึงสินค้า อาหาร และบรรยากาศร้านมากกว่า) ข้อสังเกตข้างต้นจึงเป็นการอนุมานจากบริบทของร้าน (ความหนาแน่นของลูกค้า + การมีระบบสมาชิก) มากกว่าเป็นปัญหาที่มีคนพูดถึงตรงๆ
